## Hardware and configuration

- Windows: the project owner’s Windows laptop, current Chrome or Edge, keyboard; mouse optional. Use landscape/fullscreen-sized window. Internet is required for first load.
- iPhone: project owner’s iPhone 17 Pro, current iOS/Safari, landscape orientation, direct touch. Internet is required for first load/install.
- Players/accounts: one local player; no account or sign-in.
- Save: existing local build save is acceptable. If the wild creature is already gone before testing, clear only this site’s data and restart; this permanently removes that local test save.
- Audio/controller/accessories: not required. Controller is not supported in this build.
- Locale/settings: English; default device graphics settings; no feature flags.

## Installation

### Windows

1. Open the Artifact URL in current Chrome or Edge.
2. Wait for **Mossmere Path**, **Preproduction graybox**, and **Build 0.0.7** to appear.
3. Optional PWA installation: click **Install game** when shown, or use the browser address-bar install icon, then approve installation.
4. Installation succeeds when the game opens in its own window. If WebGL is reported unavailable, update/restart the browser and confirm hardware acceleration is enabled before reporting BLOCKED.

### iPhone

1. Open the Artifact URL in Safari—not an in-app browser.
2. Tap Share, then **Add to Home Screen**, then **Add**. This does not erase or alter other apps.
3. Installation succeeds when the Aetherlings icon appears on the Home Screen.
4. Launch it and rotate the phone sideways when prompted.

## Launch

1. Use the installed icon/window, or open the exact Artifact URL.
2. Confirm the upper/right build card reads **Build 0.0.7** and eventually shows an FPS number.
3. Confirm the route label reads **Mossmere Path**. That is the starting location for Test 1.

## Test cases

### Test 1 — Windows launch, movement, and performance

- Requirement IDs: R-001, R-002, R-006
- Purpose: Prove WebGL startup, keyboard movement/collision, camera tracking, and Windows performance.
- Starting state: Build 0.0.7 at Mossmere Path on Windows, encounter creature still present.
- Steps:
  1. Hold W, A, S, and D individually; then try two-key diagonals.
  2. Walk against both route edges and the outpost obstruction.
  3. Move continuously around the route for 90 seconds while watching the FPS counter.
- Expected result:
  1. Logan moves smoothly in eight directions, faces travel direction, and the camera follows.
  2. Logan cannot leave the route or pass through the outpost obstruction; movement can slide along it.
  3. FPS is preferably near 60 with no persistent stutter.
- Pass condition: All controls respond, collision holds, camera stays stable, and FPS does not remain below 30.
- Fail condition: Blank/fallback screen, stuck input, clipping, unstable camera, or sustained sub-30 FPS.
- Evidence if failed: Screenshot plus 10–20 second video showing input/FPS.
- Cleanup/reset: Return near the route’s lower starting half.
- Result: PENDING

### Test 2 — Battle, capture, party, and healing

- Requirement IDs: R-003, R-004
- Purpose: Prove the complete current gameplay loop.
- Starting state: Wild purple/teal graybox Aetherling visible; Kivren in party.
- Steps:
  1. Approach the wild target until **Approach wild Aetherling** appears; press E.
  2. Use **Pulse Strike** at least once and note health/odds.
  3. Use **Binding Spore** and confirm **Snared** plus increased capture odds.
  4. Use **Aether Prism**. If it breaks, confirm one Prism is consumed and Mirelume counters; retry after further weakening if needed.
  5. After capture, choose **Continue with Mirelume**, open **Party**, and inspect both members.
  6. Walk north to the teal crystal, interact, and reopen Party.
- Expected result:
  1. Battle opens and movement locks.
  2. Damage lowers wild HP without reducing it below 1; odds rise as HP falls.
  3. Snare is visibly named and further improves odds.
  4. Failed captures consume a Prism and allow a counter-turn; success ends battle.
  5. Mirelume occupies slot two and Kivren retains battle damage.
  6. Crystal interaction restores all party HP and shows confirmation.
- Pass condition: The entire encounter-through-heal sequence completes with correct visible state.
- Fail condition: Soft lock, wrong counts/HP/odds, missing capture, missing party member, or healing failure.
- Evidence if failed: Screenshot of battle/party plus action sequence and counts.
- Cleanup/reset: Close the Party panel after healing.
- Result: PENDING

### Test 3 — Windows save and offline PWA

- Requirement IDs: R-005, R-007
- Purpose: Prove completed progress and position survive reload, and the installed shell launches offline.
- Starting state: Test 2 completed; two-member healed party; Windows PWA installed if supported.
- Steps:
  1. Move for at least three seconds to a recognizable position, then wait three more seconds.
  2. Refresh. Confirm **Progress restored** and inspect Party.
  3. Close every game window, reopen the installed app/icon, and inspect position/Party.
  4. After one successful online launch, disconnect Wi-Fi/Ethernet, close and reopen the installed app once, then reconnect.
- Expected result: Position, captured state, two-member healed party, and absent wild target restore; cached game shell opens offline.
- Pass condition: State survives both refresh and full reopen, and offline launch reaches Mossmere Path.
- Fail condition: New-game reset, wild target returns, party/HP changes, save-unavailable message, or offline browser error.
- Evidence if failed: Screenshot before/after plus closure method and browser/PWA mode.
- Cleanup/reset: Reconnect the laptop to the internet.
- Result: PENDING

### Test 4 — iPhone touch, layout, battle, and performance

- Requirement IDs: R-001, R-002, R-003, R-006

- Starting state: Build 0.0.7 launched from the iPhone Home Screen in landscape. A separate iPhone-local save is expected.

  2. Drag/release the left joystick in all directions and verify it recenters.
  3. Move for 90 seconds while watching FPS; record the usual and lowest sustained reading.
  4. Approach the wild target and use the circular Interact button.
  5. Complete attack, Snare, Prism capture, Party inspection, and crystal healing using touch only.
- Expected result: No primary controls are cropped; joystick releases cleanly; interaction enables only in range; battle buttons remain readable/tappable; FPS is at least 30 for most play with no sustained segment below 24.
- Pass condition: Complete touch-only loop without clipping, stuck input, crash, or sustained sub-24 FPS.
- Fail condition: Cropped/overlapping primary UI, missed taps, stuck movement, renderer failure, crash, or sustained sub-24 FPS.
- Evidence if failed: Landscape screenshot and 10–20 second screen recording with FPS visible.
- Cleanup/reset: Leave the app at Mossmere Path with Party panel closed.
- Result: PENDING
