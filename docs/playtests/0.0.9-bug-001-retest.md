## Test cases

### Test 1 — Wild health and crystal isolation

- Requirement IDs: BUG-001-A, BUG-001-B, BUG-001-C
- Purpose: reproduce the exact clarified path on the repaired build.
- Starting state: Build 0.0.9, fresh save with only Kivren, Mirelume visible, keyboard.
- Steps:
  1. Enter battle. Confirm Mirelume starts at 30/30 HP and Kivren is labeled **Your lead**.
  2. Use Pulse Strike repeatedly. Confirm Mirelume stops at 1 HP and never shows 0 HP.
  3. Let Kivren lose, choose Return, and confirm Kivren is restored at Mossmere Outpost.
  4. Re-enter battle. Confirm this new Mirelume starts at 30/30 HP.
  5. Leave through defeat again, walk to the teal crystal, interact, then re-enter battle.
  6. Confirm Mirelume again starts at 30/30 HP independently of crystal use. Weaken it to 1 HP, apply Binding Spore, confirm 100%, and capture it.
- Expected result:
  1. Wild HP never reaches 0.
  2. Every re-encounter starts wild Mirelume at 30/30 with no Snare.
  3. The crystal restores party members only; it is not responsible for wild health.
  4. The 1-HP-plus-Snare Prism captures and adds Mirelume to Party.
- Pass condition: all expected results occur without wild state carrying between encounters.
- Fail condition: wild reaches/starts at 0, prior wild damage/status carries over, crystal appears to change active wild state, or 100% capture fails.
- Evidence if failed: screenshot showing Build 0.0.9, both ownership labels, and both HP values; state the last action.
- Cleanup/reset: leave the captured party saved; do not reset again.
- Result: PENDING

### Test 2 — In-game test-save reset

- Requirement IDs: RESET-001
- Purpose: prove future test resets no longer require browser settings.
- Starting state: Build 0.0.9 with Mirelume already in Party.
- Steps:
  1. Open Party and select **Reset Test Save**.
  2. Read the permanent-deletion warning and choose Cancel once; confirm nothing reloads or changes.
  3. Select **Reset Test Save** again and approve.
  4. After automatic reload, open Party.
- Expected result:
  1. Cancel preserves progress.
  2. Approval reloads into a fresh save containing only Kivren, with wild Mirelume visible on the route.
- Pass condition: cancel is harmless and approval clears both primary and backup progress.
- Fail condition: cancel changes data, reset errors, old party returns, or wild Mirelume remains absent.
- Evidence if failed: screenshot showing Build 0.0.9 and Party/route state; include which confirmation choice was used.
- Cleanup/reset: proceed directly to Test 1 from the fresh state.
- Result: PENDING
