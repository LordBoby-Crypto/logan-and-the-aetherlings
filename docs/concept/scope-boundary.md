# Scope Boundary

## Production assumptions

AI performs design, engineering, documentation, integration, and test preparation; the user makes material decisions and performs requested device tests. Development is milestone-driven with no promised release date. Windows development leads, but iPhone is validated during meaningful milestones. Recurring services are minimized.

## In scope

The prototype contains one short route, one settlement/outpost, Logan, three temporary Aetherlings, one wild encounter, one trainer battle, exploration, collision, interaction, turn-based battle, Aether Prism capture, party management, healing, local saves, keyboard/touch controls, and PWA installation. The eventual release target is one single-player regional adventure with 60–80 original Aetherlings.

## Out of scope

Pokémon intellectual property, seamless open world, multiplayer/trading/co-op/MMO/live service, user-generated content, native App Store release during initial development, full voice acting, exploitative monetization, and 100+ launch creatures.

## Deferred

Cloud saves, accounts, native packaging, later regions, multiplayer, trading, deep character customization, broader localization, and post-release creatures.

## Unknowns requiring validation

iPhone scene budget, MeshyAI asset throughput, touch comfort, local-save durability, encounter presentation, affinity balance, and party/turn structure all require preproduction tests with explicit fallbacks.

## Difficulty and cost

- Exploration foundation: Moderate.
- Battle/capture: High.
- iPhone performance: High.
- Save system: Moderate.
- 60–80 creature production: Extreme due to design, modeling, rigging, animation, balance, audio, and QA.
- World/content production: High.
- Static hosting: Low.
- Optional cloud sync: Moderate–High.

Likely later costs include MeshyAI credits, hosting/backend if cloud sync is approved, audio/music, domain registration, and possible specialist asset cleanup. Exact totals are not established.

## Change-control rule

User approval is required for platform order, core loop, camera, multiplayer, monetization, content rating, creature target, release scope, cloud dependency, major visual tier, and high residual risks. The AI studio may decide reversible implementation details consistent with approved pillars.
